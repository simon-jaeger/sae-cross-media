CSS: Global

Externes CSS

CSS kann alternativ in einer eigenen Datei geschrieben werden, die in mehreren HTML-Dateien referenziert werden kann. Das ist nützlich, um globale Regeln zu definieren. Beginne immer mit lokalen Regeln und erstelle globale Regeln, sobald du merkst, dass du dich zu oft wiederholst. 

<link href="style.css" rel="stylesheet">

Selektoren und Regel-Blöcke

Globale Regel-Blöcke beginnnen immer mit einem Selektor, gefolgt von geschweiften Klammern. Innherhalb der Klammern werden die CSS-Regeln erfasst, die global für diese Elemente gelten sollen.

    h1 {
      font-weight: bold;
      margin-bottom: 16px;
      /* etc */
    }

Kommentare

Benutze Kommentare, um deine externen CSS-Dateien zu dokumentieren:

/* Das ist ein Kommentar in CSS */

Klassen

Regel-Blöcken können statt für vorgegebene HTML-Elemente auch für selbstdefiniert Klassen geschrieben werden. Das erlaubt das Gruppieren eines Sets an CSS-Regeln, die dann über das Attribute class einfach auf ein beliebiges Element angewendet werden können.

Div-Element mit der Klasse .card
Article-Element mit der Klasse .card 
    /* global.css */
    .card {
      padding: 24px;
      border-radius: 4px;
      margin-bottom: 16px;
      background-color: #ffffff;
      box-shadow: 0 1px 3px 0 #aaaaaa;
    }

    <!-- index.html -->
    <div class="card">
      Element mit der Klasse <em>.card</em>
    </div>
    <div class="card">
      Und noch ein Element mit der gleichen Klasse
    </div>

Pseudo-Selektoren

→YouTube

Um CSS-Regeln zu definieren, die nur unter bestimmen Bedingungen aktiv werden, kommen sogenannte Pseudo-Selektoren zum Einsatz.

    /* global.css */
    button {
      padding: 8px;
      border: 1px solid #333333;
    }
    button:hover {
      background-color: #333333;
      color: #cccccc;
    }
    input {
      padding: 8px;
      outline: 1px solid #333333;
    }
    input:focus {
      outline: 1px solid #EF4444;
    }

    <!-- index.html -->
    <button>hover me</button>
    <input type="text" value="focus me">
